After finishing the exploratory testing project, it was time to move on to test automation. My first task was to use Postman to create some basic automated tests. I was already familiar with using Postman to send manual tests to the Restful Booker API, so now it was time to automate!

I was given a brief introduction to creating tests with Postman, and then assigned homework to automate the following:

  • Create a booking that will then be deleted
  • Post a booking and confirm that the values sent are the same values returned
  • Quickly create 10 bookings

Although these tests were fairly simple, I made sure to only ever use one codified oracle per test in order to keep the intent targeted. For example when testing that the returned values matched those sent, I checked each part of the booking separately in it's own test, such as first name, last name, check in date, check out date, rather than the entire response body in one test.

Test Automation Tools

The next step was to begin looking at more complex tests, utilising new software and tools, as well as a new language - Java. To prepare, I completed the Ministry of Testing Introduction to Java and Selenium WebDriver API Introduction in Java courses.

After completing the courses, I was better prepared to tackle my larger project: using Selenium WebDriver to automate testing of the Restful Booker Platform's login and room creation forms. Everything in the project was completely new to me - Java, IntelliJ, Selenium, JUnit, so it was definitely a challenge, but it was a great feeling to finally see all my tests run! I created just a few tests, but I tried to make these targeted, so that if issues arose, it was easier to identify where the problem occurred. You can find my code here.
