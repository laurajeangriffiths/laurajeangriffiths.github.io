My main project in exploratroy testing began with assessing quality and risk. I was tasked with exploratory testing the Restful-Booker API. With 3 key quality characteristics, and the user stories, I began by conducting a risk analysis to identify possible risks and areas for exploration. After considering possible risks, I used these to draw up charters ready for testing.

Quality Characteristics Users should be able to:
Accuracy
  • Create bookings
  • View bookings by ID
  • Retrieve a booking by ID
  • Search via booking dates
  • Update bookings using ID
  • Protect bookings from changes and deletion
  • Delete booking by ID
User Friendliness
Consistency
  • Creating a booking
    • Is authentication required?
    • Formats
      • Name - min. and max. character limit?
      • Date - leap years, fake dates
  • Updating a booking
    • Is authentication required?
    • Are updates saved?
    • Are formatting rules the same as for creating a booking?
Risk Analysis of Restful Booker API
Quality Characteristic Risk
Accuracy
  • Is correct data returned?
Consistency
  • Is data always in the same format?
  • Is behaviour consistent?
Ease of use
  • Is it easy to use?
  • Is data clear?

These considerations were then used to create charters for exploratory testing of the API. The resulting mindmap can be found on the next page.
