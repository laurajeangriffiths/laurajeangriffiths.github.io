My main project in exploratory testing was to conduct testing of the Restful Booker API. I began this project with a risk analysis to identify possible risks and areas for deeper exploration. I was provided with 3 key quality characteristics, as well as the user stories, which I utilised to prompt my thinking and assessment of risk.

I began by taking the 3 quality characteristics, and considering risks around these. After considering possible risks, I used these to draw up charters ready for testing.

Following on this, I referred to the user stories as a prompt to identify areas for exploration. These findings became the basis of my testing charters.

Quality Characteristics Users should be able to:
Accuracy
  • Create bookings
  • View bookings by ID
  • Retrieve a booking by ID
  • Search via booking dates
  • Update bookings using ID
  • Protect bookings from changes and deletion
  • Delete booking by ID
User Friendliness
Consistency
  • Creating a booking
    • Is authentication required?
    • Formats
      • Name - min. and max. character limit?
      • Date - leap years, fake dates
  • Updating a booking
    • Is authentication required?
    • Are updates saved?
    • Are formatting rules the same as for creating a booking?

These considerations were then used to create charters for exploratory testing of the API. The resulting mindmap can be found on the next page.

Risk Analysis of Restful Booker API
